# cryptopia 웹 사이트는 약간의 지연 후 라이브 간다, 사전 해킹 상태에서 개최 사용자 잔액 ...

###### 2019-03-05 08:03

최근 해킹 암호 교환 크립 토 피아는 읽기 전용 형식으로 자사의 웹 사이트를 다시 시작 했다, 그 사전 해킹 상태로 사용자 잔액을 재개와 함께, 3 월에 트 윗 거래소 5.

트 윗의 최신 시리즈에서, cryptopia는 더 계산 리베이트의 기초로 1 월 14에서 개최 고객의 잔액을 사용 하는 것을 확인 했다.

에 따라 $1600만 1 월 중순에 해킹, 크립 토 피아는 여전히 유지 보수 모드에 남아 있었다 후 뉴질랜드 경찰은 2 월에 작업을 재개 할 준비가 된 것으로 알려졌다 13.

뉴스에 응답, cryptoptia는 교환이 완전히 자신의 손실을 식별 하 고 균형이 완전히 안전 하 게 보장 할 때까지 거래를 재개 할 수 없다고 설명 했다.

1 월 중순에 해킹의 초기 보고서 후, 추가 증거는 해커가 나중에 2 주 후반으로 교환에서 몰래 암호화 했다 표면화.

3 월 3 일에 다른 exchange 취약점 뉴스에서 monero (xmr)에서 파생 된 암호 프로젝트 인 료 (료)는 몬에로 지갑 소프트웨어의 버그가 암호화 거래소에 가짜 예금을 사용할 수 있다고 주장 하는 게시물을 게시 했습니다.

설명 된 취약점, 곧 릴리스에서 패치되도록 설정 되어, 잠재적으로 해커는 교환에 xmr의 임의의 금액의 보증금을 가짜 수 있습니다.
