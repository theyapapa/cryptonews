# ' 블록 체인 산적 ': 해커는 약한 개인 키를 추측 하 여 ETH의 수백만 가치를 훔치는 방법 ...

###### 2019-04-28 06:04

이번 주 초, 미국 기반 보안 컨설팅 회사 독립 보안 평가자 (ISE)는 Ethereum 블록 체인에 대 한 개인 키에 대 한 보고서를 게시 했습니다.

"그것은 당신이 당신의 사용자 이름과 암호를가지고 있는 당신의 은행 로그인과는 다르다 \ [...] 따라서 두 명의 개인 들이 뇌 지갑을 만드는 데 동일한 암호를 사용 하는 경우, 즉 개인 키를 생성 하는 과정에서 암호 수반 되는 지갑은 ' password123 '-그들은 모두 똑같은 지갑을 소유할 것입니다. "

## 블록 체인 산적

위에서 언급 한 바와 같이, ISE 팀은 개인 키와 관련 된 지갑 중에서 최적이 아닌 방법으로 발견 된 일부 지갑이 특정 주소로가는 많은 트랜잭션을가지고 있었고 돈이 다시 오지 않는 방법을 발견 했습니다.

"블록 체인은 의도 대로 작동 하 고 있으며, 사람들이 그것을 사용 하는 방식에 불과합니다" 라고 그는 연구를 하는 동안 자신의 팀이 직면 한 윤리 관련 문제를 설명 했다.

"이에 대 한 연구를 시작 하기 전에 백만 달러의 키를 가진 지갑을 찾으면 우리는 윤리적 딜레마를 가졌습니다.

예를 들어 ISE 보고서는 가장 인기 있는 취약 한 개인 키 중 하나가 패리티 지갑을 사용 하 여 빈 복구 구문 (즉, "")에서 생성 된 것 이라고 설정 했습니다.

"잠시 동안 패리티를 사용 하면 기본 암호를 사용할 수 없으며 그에 따라 개인 키를 생성 하 게 됩니다." 얀는 지갑 개발자가 어떤 시점에서 문제를 해결 했다고 주장 하는 추가 설명 합니다.

얀에 따르면 지갑 제작자는 ISE에 아직 연락을 하지 않았습니다.

"그건 재미 있는 문제 때문에 어떤 지갑 책임이 있었다, 지갑 전혀-그것은 단지 사람들이 잘못 된 개인 키를 입력 될 수 있습니다., 그것은 지갑의 초기 디버그 버전 수 있습니다., 그것은 개발자 자신 수 있습니다.
