# 스텔라의 블록 체인이 오프 라인으로 전환 되어 프로젝트가 분권 화 부족을 확인 합니다. ...

###### 2019-05-20 13:05

즉, 전체 네트워크를 사용 하 여 비트 코인과 같은 트랜잭션의 유효성을 검사 하는 대신 스텔라는 소위 쿼럼 슬라이스 (각 유효성 검사기 노드가 동의 하도록 선택 하는 노드 집합)에 의존 합니다.

별의 개발과 채택에 전념 하는 비영리 단체인 스텔라 개발 재단은 "새로운 노드가 너무 빨리 합의에 대 한 책임을 너무 많이 맡게 되었기 때문에 네트워크가 붕괴 되었다고 믿습니다."

특히, 연구원은 전체 항성 네트워크가 제한 된 양의 노드 (주로 SDF 자체에 의해 제어 되는 것)에 달려 있다고 강조 했습니다.

"우리는 스텔라의 모든 노드가 두 개의 노드만 실패할 경우 스텔라 컨센서스 프로토콜을 실행할 수 없다는 것을 보여 줍니다."

사후 분석에서 SDF는 너무 많은 새 노드가 더 분산 되도록 입찰에 추가 되었기 때문에 네트워크가 동결 된 것으로 설명 했습니다.

"우리는 스텔라가 ' 오버 중앙 집중화 ' 라고 주장 했 고, 어떻게 든 SDF의 노드에 오류가 발생 하 여 전체 네트워크를 끌어 올 렸 습니다.

특히 김은 논문을 제출 했을 때 세 개의 SDF 검사기 노드 중 2 개가 오프 라인이 된 경우 스텔라 네트워크가 붕괴 되는 것을 Cointelegraph에 게 말했습니다.

네트워크 장애의 여파를 해결 하 고 다시 온라인 상태로 전환 하기 위해 SDF는이 세 개의 유효성 검사기를 모두 김에 따라 쿼럼 집합에 포함 하 고 3 개의 SDF 유효성 검사기 노드 중 2 개가 아래로 이동 하는 경우 "1 단계로 돌아가기"를 반환 했습니다. , 스텔라 네트워크가 붕괴 됩니다.

— 카일 맥 콜 👨🏻 🚀 (@kylemccollom) 2019 년 5 월 16 일 유사 하 게, 별의 하위 reddit에 사용자 게시물 마찬가지로 SDF 노드가 아래로 갔다 때문에 네트워크 합의에 도달 할 수 없는 암시, 맥 calleb에 의해 거부 된 코멘트 섹션: 스텔라의 공동 설립자 "SDF 노드와 실제로 네트워크의 대부분의 유효성 검사기는 여전히 가동 되었지만" 쿼럼에 있는 충분 한 노드에서 듣지 않았기 때문에 원장을 안전 하 게 닫을 수 없습니다. "
